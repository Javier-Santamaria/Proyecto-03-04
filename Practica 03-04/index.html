Hosting compartido

CONCEPTO

Este tipo de servidor consta de un solo hardware donde los programadores pueden guardar o publicar sus paginas web, aunque tambien depende de la capacidad de los recursos que tenga el hosting.

VENTAJAS

Alojamiento web economico:
Es el más economico,  ya que aloja a miles de paginas web a la vez. pero con limitaciones.
Administrar y m antenimiento:
Los que otorgan hosting compartidos se encargan de su mantenimiento por lo que el programador se despreocupa.
Certificación SSL:
Las empresas que ofrecen hosting dan la oportunidad de obtener un certificado web o SSL, el cual da a tu pagina web mayor seguridad y accesibilidad.

FUNCIONAMIENTO

Este modelo de hosting funciona con un conjunto de paginas web tienen acceso compartido a recursos del servidor. En este caso, la capacidad de almacenamiento es dividida entre todos los usuarios, incluyendo RAM, CPU y la máquina física.

SERVIDORES DEDICADOS

CONCEPTO

Este tipo de servidor consta de adquirir todo un servidor dedicado a tu pagina web, donde no necesitará compartir recursos con otra página web.

FUNCIONAMIENTO
VENTAJAS

Más confiable:
Con respecto al desempeño de la página podrá verse la rapidez al cargar tanto el texto, las imagenes, las acciones que realices serás más rápidas, etc.
Control total:
Significa que serás quién tenga el control sobre la gestión, administración, actualizaciones de la página, etc.
Mayor seguridad:
Ya que no compartes el servidor con otras paginas web, la seguridad de tu página será mayor aunque ten en cuenta que debes estar al tanto de darle alguna revisión eventualmente.

Es un servidor físico que ofrece servicios a un unico inquilino. El inquilino del servidor puede optimizarlo según sus necesidades para obtener un mayor nivel de desempeño, seguridad y confiabilidad.

SERVIDORES CLOUD

CONCEPTO

Este tipo de servidor consta de darte hospedaje a tus paginas web en multiples servidores en la nube.

FUNCIONAMIENTO

Los servidores de nube son posibles gracias a la virtualizacion. Se instala un software de gestión llamado hipervisor en servidores físicos para conectarlos y visualizarlos. Sus recursos combinados se desvinculan y se agrupan para crear servidores virtuales.
